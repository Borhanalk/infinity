# تعليمات إنشاء المسؤول

## الخطوات السريعة:

### 1. إنشاء ملف `.env.local`

في جذر المشروع، أنشئ ملف `.env.local` وأضف:

```env
ADMIN_EMAIL=admin@infinity.com
ADMIN_PASSWORD=A@123
JWT_SECRET=change-this-to-a-strong-secret-key-minimum-32-characters
```

⚠️ **مهم:** استبدل `JWT_SECRET` بمفتاح قوي (32 حرف على الأقل)

### 2. إنشاء Migration

```bash
npm run prisma:migrate
```

أو:

```bash
npm run db:push
```

### 3. إنشاء المسؤول

```bash
npm run create-admin
```

---

## ما سيحدث:

✅ سيتم **حذف جميع المسؤولين السابقين** تلقائياً
✅ سيتم إنشاء مسؤول جديد بالبيانات المحددة
✅ كلمة المرور ستكون **مشفرة** باستخدام bcrypt
✅ لن يتم عرض كلمة المرور في أي مكان

---

## الأمان:

- ✅ كلمات المرور **مشفرة** في قاعدة البيانات
- ✅ لا يمكن استرجاع كلمة المرور الأصلية
- ✅ جميع البيانات الحساسة في `.env.local` (غير متضمنة في Git)
- ✅ استخدام JWT tokens للتحقق من المسؤولين

---

## التحقق:

بعد إنشاء المسؤول، يمكنك التحقق من خلال:

```bash
npm run prisma:studio
```

افتح جدول `Admin` للتحقق من البيانات.

---

## ملاحظات:

- إذا نسيت كلمة المرور، يجب إنشاء مسؤول جديد
- يمكنك تعديل `scripts/create-admin.ts` لإضافة مسؤولين إضافيين
- في الإنتاج، استخدم كلمة مرور قوية جداً
